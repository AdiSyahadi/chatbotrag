Embed Widget

Integrasikan chatbot ke website manapun dengan satu baris kode.

Preview Widget


Widget akan muncul di pojok kanan bawah halaman ini.

Cara Embed

Tambahkan tag <script> berikut sebelum penutup </body> di website Anda:

<script src="/static/widget.js" data-server=""></script>
Catatan:
  • Ganti URL data-server dengan alamat server produksi Anda jika di-deploy.
  • Widget secara otomatis membuat tombol chat melayang di pojok kanan bawah.
  • Tidak perlu CSS atau dependensi tambahan — semua sudah termasuk di dalam script.
  • CORS sudah diaktifkan, sehingga bisa dipanggil dari domain manapun.
Public API Reference

GET /api/v1/health

Cek status server.

// Response
{ "status": "ok" }

POST /api/v1/chat

Kirim pertanyaan dan dapatkan jawaban dari RAG pipeline.

Content-Type: application/json  |  Auth: Tidak perlu (public endpoint)

Parameter Tipe Wajib Keterangan
message string ✅ Ya Pertanyaan atau pesan dari user
session_id string Opsional ID unik per sesi percakapan. Jika dikirim, chatbot akan mengingat konteks percakapan sebelumnya dalam sesi yang sama. Gunakan UUID atau string acak unik per user/session. 
// Request (minimal)
{
  "message": "Apa itu RAG?"
}

// Request (dengan memory percakapan)
{
  "message": "Berapa biaya kuliahnya?",
  "session_id": "user-123-abc"
}

// Response
{
  "reply": "RAG (Retrieval-Augmented Generation) adalah ..."
}
Penting: Pastikan dokumen sudah di-upload dan diproses sebelum menggunakan endpoint chat. Tanpa data di vectorstore, chatbot tidak bisa menjawab pertanyaan berbasis dokumen.

HTTP Status Codes

Status Arti Kapan terjadi
200 Berhasil Pertanyaan berhasil dijawab
400 Bad Request Message kosong atau format JSON salah
500 Server Error Gagal memproses pertanyaan (misal: vectorstore kosong)
503 Service Unavailable API key Gemini belum dikonfigurasi oleh admin

Format Error Response

Semua error mengembalikan JSON dengan field error:

// 400 — Message kosong
{ "error": "Message tidak boleh kosong." }

// 500 — Gagal proses
{ "error": "Gagal memproses pertanyaan." }

// 503 — Belum dikonfigurasi
{ "error": "Service belum dikonfigurasi." }

Tip: selalu cek response.ok atau status_code sebelum membaca reply.

Batasan

Item Nilai Keterangan
Session timeout 30 menit Session otomatis dihapus setelah 30 menit tidak aktif
Maks. history per session 20 pesan Pesan lama otomatis dihapus jika melebihi batas
Rate limit Tidak ada Tergantung kuota Gemini API key yang dipakai
Auth Tidak perlu Public endpoint, tidak butuh API key dari sisi pemanggil
Contoh cURL
curl -X POST /api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Apa jurusan yang tersedia?", "session_id": "user-123"}'
Contoh Integrasi Framework

Laravel (PHP)

// Di controller Laravel kamu
use Illuminate\Support\Facades\Http;

public function chat(Request $request)
{
    $response = Http::post('/api/v1/chat', [
        'message'    => $request->input('message'),
        'session_id' => session()->getId(),  // otomatis unik per user
    ]);

    return response()->json([
        'reply' => $response->json('reply'),
    ]);
}

JavaScript (Fetch API)

// Bisa dipakai di React, Vue, Svelte, Next.js, dll.
const SESSION_ID = crypto.randomUUID();

async function chat(message) {
  const res = await fetch('/api/v1/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ message, session_id: SESSION_ID }),
  });
  const data = await res.json();
  return data.reply;
}

Python (Requests)

import requests, uuid

session_id = str(uuid.uuid4())

response = requests.post('/api/v1/chat', json={
    'message': 'Apa jurusan yang tersedia?',
    'session_id': session_id,
})
print(response.json()['reply'])
